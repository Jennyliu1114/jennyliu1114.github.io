    33.搜索旋转排序数组

    整数数组 nums 按升序排列，数组中的值 互不相同 。

    在传递给函数之前，nums 在预先未知的某个下标 k（0 <= k < nums.length）上进行了 向左旋转，使数组变为 [nums[k], nums[k+1], ..., nums[n-1], nums[0], nums[1], ..., nums[k-1]]（下标 从 0 开始 计数）。例如， [0,1,2,4,5,6,7] 下标 3 上向左旋转后可能变为 [4,5,6,7,0,1,2] 。

    给你 旋转后 的数组 nums 和一个整数 target ，如果 nums 中存在这个目标值 target ，则返回它的下标，否则返回 -1 。

    你必须设计一个时间复杂度为 O(log n) 的算法解决此问题。

    示例 1：

    我的思路：
    1. 每次将数组一分为二：取中点 mid，那么 [left, mid] 和 [mid+1, right] 中至少有一个是有序的（因为原数组是升序且无重复）。
    2. 判断哪一半是有序的：
      • 如果 nums[left] <= nums[mid] → 左半部分有序
      • 否则 → 右半部分有序
    3. 检查 target 是否落在有序的那一半：
      • 如果在，就在该区间做标准二分查找（缩小范围）
      • 如果不在，就去另一半（无序部分）继续递归/迭代

    153.寻找旋转排序数组中的最小值

    已知一个长度为 n 的数组，预先按照升序排列，经由 1 到 n 次 旋转 后，得到输入数组。例如，原数组 nums = [0,1,2,4,5,6,7] 在变化后可能得到：

    • 若旋转 4 次，则可以得到 [4,5,6,7,0,1,2]
    • 若旋转 7 次，则可以得到 [0,1,2,4,5,6,7]

    注意，数组 [a[0], a[1], a[2], ..., a[n-1]] 旋转一次 的结果为数组 [a[n-1], a[0], a[1], a[2], ..., a[n-2]] 。

    给你一个元素值 互不相同 的数组 nums ，它原来是一个升序排列的数组，并按上述情形进行了多次旋转。请你找出并返回数组中的 最小元素 。

    你必须设计一个时间复杂度为 O(log n) 的算法解决此问题。

    我的思路：

    在整个 while 循环过程中，数组的最小值 never leaves the interval [left, right]。只要这个不变量成立，那么当区间缩小到只有一个元素（left == right）时，那个元素必然就是最小值

    二分查找的作用相当于是每次都排除一半区间，直到找到一个left==right的地方。所以while那里不能写等于号！

    59.螺旋矩阵

    给你一个正整数 n ，生成一个包含 1 到 n^2 所有元素，且元素按顺时针顺序螺旋排列的 n x n 正方形矩阵 matrix 。

    思路：

    模拟顺时针画矩阵的过程:

    • 填充上行从左到右
    • 填充右列从上到下
    • 填充下行从右到左
    • 填充左列从下到上

    由外向内一圈一圈这么画下去。每画一条边都要坚持一致的左闭右开的原则
